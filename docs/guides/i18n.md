# Internationalization (i18n)

Queopius Sentinel dashboard supports language switching through the host app locale flow.

Current locale set:

- ES
- EN
- FR
- PT-BR (`pt` locale key)

## How it works

- Locale switch endpoint: `/locale/{locale}`
- Session stores selected locale
- Sentinel UI reads translated labels from app lang files (`lang/<locale>/sentinel.php`)

## Required app setup

- Include locales in `config/app.php`:

```php
'supported_locales' => ['es', 'en', 'fr', 'pt'],
```

- Ensure locale middleware runs on web requests.

## Translation files

Add/edit dashboard labels in:

- `lang/en/sentinel.php`
- `lang/es/sentinel.php`
- `lang/fr/sentinel.php`
- `lang/pt/sentinel.php`

## Add a new locale

1. Add locale code to `supported_locales`.
2. Create `lang/<new-locale>/sentinel.php`.
3. Add language option label in Sentinel dashboard locale selector if needed.
4. Clear cache:

```bash
php artisan optimize:clear
```
